分析总览

这里是启动前摘要。真正填写内容时，使用左侧目录切到对应设置区块。

01

参数定位

输入参数名、位置、用途说明或接口上下文，帮助大模型明确这次分析的真实目标。

02

JS 收集

支持多文件上传、zip 包导入、网页 URL 抓取与外部 JS URL 列表。

03

样本校验

可选提供已知明文与密文，用于对推断出的流程做自动一致性验证。

04

提示词策略

支持固定复用系统提示词与操作提示词，持续约束模型按你的逆向分析方法输出结构化结论。

05

DeepSeek 主分析

当前会先做全量通读，再做重点精读，最后结合结果复核压低分析抖动。

参数上下文

参数名是强制项。其余字段不是必填，但会显著影响静态排序结果。

JS 来源

上传模式稳定性最高。网页 URL 模式只抓取当前页面直接引用 JS 与静态可发现 chunk。
